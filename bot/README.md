# בוט וואטסאפ — צילום תדמית ופורטרט · שרון גבאי

"המוח" של הבוט + **מצב בדיקה מקומי בטרמינל**.
בשלב הזה אין חיבור לוואטסאפ, אין יומן אמיתי ואין מפתחות — רק לוגיקת השיחה,
תמחור, וקביעת מועד מדומה, עם נקודות חיבור מסומנות לעתיד.

## הרצה

דורש Node.js 18+ (אין תלויות חיצוניות).

```bash
cd bot

# שיחה אינטראקטיבית מלאה בטרמינל:
npm run bot

# הדגמה אוטומטית של 5 תרחישים (בלי הקלדה):
npm run demo

# בדיקות אוטומטיות של חוקי התמחור והזמינות:
npm test
```

במצב האינטראקטיבי עונים על שאלות הבוט (אפשר מספרים או טקסט חופשי).
להפסקה: מקלידים `יציאה`.
לקיבוע "היום" לצורך הדגמה דטרמיניסטית: `BOT_TODAY=2026-06-21 npm run bot`.

## מהלך השיחה

1. **ברכה וסינון** — סוג הצילום, מתי, ואיפה (בסטודיו / אצל הלקוח).
2. **בכל שלב** — הצעה לעבור לשיחה אישית עם שרון (מקלידים `שרון` או `טלפון`).
3. **חבילה ומחיר** — הצגת המחירון לפי המיקום, עם הדגשת **סטנדרט** כמומלצת.
4. **תיאום תאריך** — הבוט מציע מועדים פנויים לפי חוקי הזמינות.
5. **קביעת מועד** — ביומן בדיקה מדומה (בזיכרון).
6. **התראה לשרון** — תמיד, עם כל פרטי הפנייה (בשלב זה: הדפסה ללוג).

### מקרים מיוחדים

- **חברה גדולה (הדשוטס לעובדים רבים)** — הבוט לא קובע מועד לבד, אלא מתאם שיחת טלפון.
- **צוות / קבוצה** — מציג תמחור לפי אדם (400–690 ₪), ומחבר לשרון לבניית הצעה מדויקת.
- **צילום אצל הלקוח** — חוסם יום שלם; שאר הפרטים נסגרים בין שרון ללקוח בנפרד.

## חוקי תמחור וזמינות

המחירון נמצא ב-`src/config/pricing.js` וחוקי הזמינות ב-`src/config/availability.js`
(ימים א׳–ה׳, 9:30–13:30; משכי סטודיו: בסיס 30 / סטנדרט 60 / פרימיום 90 דק׳, עם רווח 30 דק׳).

## מבנה הקוד

```
bot/
├── src/
│   ├── brain.js                    # "המוח" — מכונת מצבים של השיחה (עצמאי מהערוץ)
│   ├── messages.js                 # כל טקסטי הבוט בעברית (טון מקצועי וחם)
│   ├── calendar.js                 # יומן בדיקה מדומה + לוגיקת הזמינות
│   ├── cli.js                      # מצב הבדיקה האינטראקטיבי בטרמינל
│   ├── demo.js                     # הדגמה אוטומטית של תרחישים
│   ├── config/
│   │   ├── pricing.js              # מחירון (מקור אמת יחיד)
│   │   └── availability.js         # חוקי ימים/שעות/משכים/רווחים
│   └── integrations/               # >>> נקודות חיבור לעתיד <<<
│       ├── whatsapp.js             # חיבור וואטסאפ (placeholder)
│       ├── google-calendar.js      # חיבור יומן גוגל אמיתי (placeholder)
│       └── notify.js               # התראה לשרון (כרגע: לוג; בעתיד: אמיתית)
└── test/
    └── run.js                      # בדיקות אוטומטיות
```

## נקודות חיבור לעתיד

כל מקום שמיועד לחיבור אמיתי מסומן בקוד בהערה `>>> נקודת חיבור עתידית <<<`:

- **וואטסאפ** (`src/integrations/whatsapp.js`) — קבלת הודעות ושליחתן.
  כל מספר טלפון יקבל מופע `Brain` משלו (session). ה-`Brain` כבר עצמאי מהערוץ.
- **יומן גוגל** (`src/integrations/google-calendar.js`) — לממש מחלקה עם אותו
  ממשק כמו `MockCalendar` (קריאת זמינות + קביעה רק בזמן פנוי) ולהזריק ל-`Brain`.
- **התראה לשרון** (`src/integrations/notify.js`) — להחליף את ההדפסה ללוג
  בשליחה אמיתית (וואטסאפ/מייל/SMS), באותה חתימה `notifySharon(lead)`.
