# בוט וואטסאפ — צילום תדמית ופורטרט · שרון גבאי

"המוח" של הבוט + חיבור **Green API** (וואטסאפ) + **הפעלה ידנית** + מצב בדיקה מקומי.
היומן הוא עדיין בדיקה מדומה, וההתראה לשרון מודפסת ללוג — יומן גוגל ומפתח קלוד
יחוברו בנפרד בהמשך.

## הפעלה ידנית (חשוב)

ברירת המחדל: **הבוט שקט ולא עונה לאף אחד.**

- שרון שולח את המילה **"בוט"** *מהמכשיר שלו* בתוך שיחה → הבוט נכנס ומנהל את אותה
  שיחה בלבד (מברך ומתחיל לתאם).
- שרון שולח **"סיום"** מהמכשיר שלו → השליטה חוזרת אליו, הבוט יוצא מהשיחה.
- הזיהוי הוא של **הודעה יוצאת מהמכשיר של שרון** — לא של הודעת לקוח, ולא של הודעות
  שהבוט עצמו שלח (כדי שלא ייווצרו לולאות).

המתג היחיד להפיכת הבוט לאוטומטי שעונה לכולם (בלי לגעת בקוד):
```
BOT_AUTO_REPLY_ALL=true
```

## הרצה

דורש Node.js 18+ (אין תלויות חיצוניות).

```bash
cd bot

# שיחה אינטראקטיבית מלאה בטרמינל:
npm run bot

# הדגמה אוטומטית של 5 תרחישים (בלי הקלדה):
npm run demo

# בדיקות אוטומטיות של חוקי התמחור והזמינות:
npm test
```

במצב האינטראקטיבי עונים על שאלות הבוט (אפשר מספרים או טקסט חופשי).
להפסקה: מקלידים `יציאה`.
לקיבוע "היום" לצורך הדגמה דטרמיניסטית: `BOT_TODAY=2026-06-21 npm run bot`.

### בדיקת ההפעלה הידנית (בלי רשת)

```bash
npm run sim
```
מדמה אירועי Green API: לקוח כותב לפני הפעלה (הבוט שותק), שרון שולח "בוט"
(הבוט נכנס ומברך), הלקוח מתכתב, ואז "סיום" (הבוט שותק שוב).

## חיבור Green API והעלאה לאוויר (24/7)

הסודות מגיעים ממשתני סביבה בלבד — **שום מפתח לא נשמר בקוד**:

| משתנה | חובה | ברירת מחדל | תיאור |
|---|---|---|---|
| `GREENAPI_API_TOKEN` | ✅ | — | ה-apiToken מ-Green API |
| `GREENAPI_ID_INSTANCE` | | `7107658538` | מזהה ה-instance |
| `BOT_AUTO_REPLY_ALL` | | `false` | `true` = הבוט עונה לכולם |

יש שתי דרכי ריצה — בוחרים אחת:

### א) Vercel (webhook) — מומלץ כשהפרויקט כבר מחובר ל-Vercel

1. להגדיר את `GREENAPI_API_TOKEN` כמשתנה סביבה בפרויקט Vercel (Production).
2. לפרוס את תיקיית `bot/` כפרויקט Vercel (Root Directory = `bot`).
   נוצרת נקודת קצה: `https://<APP>.vercel.app/api/webhook`.
3. לחבר את Green API לכתובת הזו (כולל הפעלת התראות יוצאות):
   ```bash
   GREENAPI_API_TOKEN=*** node scripts/setup-greenapi.js --mode webhook \
     --url https://<APP>.vercel.app/api/webhook
   ```
4. בדיקת בריאות: פתיחת `https://<APP>.vercel.app/api/webhook` בדפדפן (GET)
   מחזירה JSON עם `tokenConfigured: true`.

### ב) עובד רץ 24/7 (polling) — אמין במיוחד לשיחות מרובות-שלבים

מתאים ל-Render Worker / Railway / Fly.io / VPS / Docker (כולל `Dockerfile` ו-`Procfile`).
שומר מצב בזיכרון התהליך — פשוט ויציב.

```bash
GREENAPI_API_TOKEN=*** node scripts/setup-greenapi.js --mode poll   # פעם אחת
GREENAPI_API_TOKEN=*** npm run worker                                 # הרצה רציפה
```

## אחסון מצב מתמיד (כדי ששיחה פעילה לא תישכח)

כברירת מחדל המצב נשמר בזיכרון של מופע ה-serverless, שעלול להתאתחל. כדי שמצב
שיחה פעילה יישמר תמיד (גם אחרי הפסקה ארוכה), מחברים **Vercel KV / Upstash Redis**:

1. בפרויקט ה-Vercel של הבוט → לשונית **Storage** → **Create / Connect Database**
   → בוחרים **Upstash for Redis** (Vercel KV) → **Connect to Project**.
2. זה מזריק אוטומטית את משתני הסביבה (`KV_REST_API_URL`, `KV_REST_API_TOKEN`).
   אין צורך לגעת בקוד.
3. Vercel יפרוס מחדש; מאותו רגע המצב מתמיד.

הבדיקה: פתיחת `/api/webhook` (GET) מחזירה `stateStore: "redis"` כשהאחסון פעיל,
או `"memory"` כשלא. הקוד נופל אוטומטית לזיכרון אם האחסון לא זמין — בלי לשבור כלום.

> לשיחות מרובות במקביל / עומס גבוה במצב אוטומטי — אפשר גם להריץ את העובד (ב').

## חיבור יומן גוגל (אופציונלי)

הבוט קורא זמינות מיומן גוגל של שרון, קובע רק בזמן פנוי, יוצר אירוע עם כל הפרטים,
ושולח זימון לאימייל של הלקוח. אם היומן לא מחובר — הבוט עובד עם יומן בדיקה ולא נשבר.

השיטה (הכי פחות הקמה): **Service Account**. צעדים לשרון:
1. ב-Google Cloud Console → פרויקט → **Enable** ל-Google Calendar API.
2. **Create Service Account** → צור לו **JSON key** (הורד את הקובץ).
3. ביומן Google של שרון → Settings → "Share with specific people" → הוסף את כתובת ה-
   Service Account עם הרשאת **"Make changes to events"**.
4. ב-Vercel → Environment Variables, הוסף:
   `GOOGLE_CLIENT_EMAIL`, `GOOGLE_PRIVATE_KEY` (התוכן מה-JSON, עם `\n`),
   `GOOGLE_CALENDAR_ID` (כתובת היומן של שרון).
5. Deploy מחדש. בדיקה: `/api/webhook` (GET) יחזיר `calendar: "google"`.

## מהלך השיחה

הבחירות הן לפי מספרים (הלקוח בוחר); רק פרטי הקשר בסוף מוקלדים.

1. **פתיחה** — ברכה, קישורי פורטפוליו/ביקורות, ובחירה: אדם אחד / מספר עובדים.
   בכל שלב אפשר לבקש לדבר עם שרון ("שרון" / "טלפון").
2. **אדם אחד** → מיקום (סטודיו / בית העסק) → חבילה ומחיר (סטנדרט מומלצת) → מועד.
   - סטודיו: מועד לפי משך החבילה (30/60/90 דק' + רווח 30), א'–ה' 9:30–13:30.
   - בית העסק: סוגר יום שלם.
3. **מספר עובדים** (תמיד בבית העסק) → מספר עובדים → תמחור לפי מדרגה + 1,500 ₪ הגעה
   והקמה → מועד יום מלא. מעל 40 עובדים → הצעה אישית של שרון (ללא תמחור/קביעה).
4. **פרטי קשר** (מוקלדים): שם, טלפון, אימייל; בבית העסק גם כתובת מלאה.
5. **קביעת מועד** ביומן (גוגל אם מחובר, אחרת בדיקה) + **אישור** ללקוח עם מדיניות ביטולים.
6. **התראה לשרון** עם כל פרטי הפנייה (כרגע: לוג).

## נקודות חיבור לעתיד

כל מקום שמיועד לחיבור אמיתי מסומן בקוד בהערה `>>> נקודת חיבור עתידית <<<`:

- **וואטסאפ** (`src/integrations/whatsapp.js`) — קבלת הודעות ושליחתן.
  כל מספר טלפון יקבל מופע `Brain` משלו (session). ה-`Brain` כבר עצמאי מהערוץ.
- **יומן גוגל** (`src/integrations/google-calendar.js`) — לממש מחלקה עם אותו
  ממשק כמו `MockCalendar` (קריאת זמינות + קביעה רק בזמן פנוי) ולהזריק ל-`Brain`.
- **התראה לשרון** (`src/integrations/notify.js`) — להחליף את ההדפסה ללוג
  בשליחה אמיתית (וואטסאפ/מייל/SMS), באותה חתימה `notifySharon(lead)`.
